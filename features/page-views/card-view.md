# Card view

*The group view.*

**Route:** `card/{listId}`
**Group:** Core loop
**Status:** active
**Last reviewed:** 2026-06-08
**Index:** [Page views](../PAGE_VIEWS.md)

What a user expects to see or do here:

- See one person at a time: avatar, name, due/ahead status
- Know when they usually answer (heat strip) and key stats (last called, avg length, pickup)
- Call them with the primary button
- Swipe/defer: left = later, right = sooner; or skip
- "View details" to open the full contact
- Top-right hamburger opens a menu: Browse people / Add contacts / Edit list
- Honest empty states ("No one is up next") with "Go home"
