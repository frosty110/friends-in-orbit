## Repo-form release notes

### Orbit 1.0.0 — first release

**Core loop**
- Card View: one contact at a time, with photo, last-call context, and list source. Tap to call via the phone dialer. Swipe left to defer, swipe right to surface sooner. Physical card tilt on drag with haptic feedback on commit.
- Automatic call detection: call history is read from the phone and used to update contact timing without manual logging. Manual log fallback available if the permission is denied.
- Home screen: mood-based list tiles with due-count badges. "Surprise me" surfaces a contact across all lists.

**Lists and rules**
- Create lists with any name. Three rule templates: spaced repetition, round robin, recency queue. Tunable parameters per list. Per-contact overrides for contacts on two or more lists.

**Contact detail**
- Full call history, notes, stats (average length, usual times, days since last call).
- Post-call note entry.
- Pause (1 week, 1 month, indefinite), ignore (hide from surfacing without deleting history), archive.

**Browse**
- Sortable, filterable full-list view. Multi-select actions: add to list, pause, ignore, move.

**Settings and data**
- Manual encrypted JSON export.
- Ignored contacts view with per-contact restore.
- Digest hour setting, per-list active hours.

**Notifications**
- Daily digest with due count.
- Per-list nudge notifications at configured times.
- Incoming call follow-up: if someone calls you, Orbit asks whether you want to call back.

**Widgets**
- 2×2 home screen widget: next contact, tap to call.
- 4×2 home screen widget: next contact plus alternatives.

**Privacy**
- Local-only storage, no servers, no accounts, no analytics.
- SQLCipher-encrypted database (contacts metadata, call history, notes).
- Three permissions: read call history, read contacts, post notifications. No internet permission.
